# GameRelay — استضف سيرفرات الألعاب من بيتك بدون فتح منافذ

نظام Relay خفيف ومفتوح المصدر ومستضاف ذاتياً يكشف سيرفرات الألعاب التي تعمل
على جهازك للإنترنت عبر ريلاي صغير تشغّله على VPS. جهازك يتصل *خارجاً* بالريلاي،
فلا تُفتح أي منافذ في راوتر بيتك — يعمل خلف CGNAT وشبكات الجوال والمزوّدات
الصارمة، ويبقى عنوان بيتك مخفياً. بديلك الشخصي المفتوح لخدمات الأنفاق المدفوعة:
بدون حسابات، بدون اشتراكات، بدون حدود.

**English: [README.md](README.md)**

```
   اللاعبون  ───────────▶   ريلاي الـ VPS   ───────────▶   جهازك في البيت
    يدخلون IP:port         (عنوان عام،              (سيرفر اللعبة على
                           Go، ~3MB)                 127.0.0.1:25565…)

   جهازك يتصل خارجاً بالريلاي — لا منافذ راوتر، وعنوان بيتك مخفي
```

## المميزات

- **أنفاق TCP و UDP** — ماينكرافت جافا وبيدروك، بال وورلد، تيراريا، FiveM،
  رست، فالهايم، وأي خدمة TCP/UDP عامة (قوالب جاهزة).
- **إعداد السيرفر بنقرة واحدة** — تطبيق Windows يثبّت ويهيّئ الريلاي على الـ
  VPS نيابةً عنك عبر SSH: يرفع الثنائية، يشغّل المثبّت، يفتح الجدار الداخلي،
  ويعبّئ السر تلقائياً. ومسار يدوي موجّه متوفر أيضاً.
- **واجهة Windows + عميل Linux** — تطبيق WinUI 3 أنيق وعميل Linux بثنائية
  واحدة (x64 و ARM64) بنفس البروتوكول والقوالب وصيغة الإعدادات.
- **بدون فتح منافذ** — جهازك يتصل خارجاً بالـ VPS، واللاعبون يتصلون بعنوانه العام.
- **إعادة اتصال تلقائية** — تراجع تدريجي ومراقب نبضات، والأنفاق تعيد فتح نفسها
  بعد الانقطاع؛ واتصالات اللاعبين النشطة تنجو من انقطاعات قناة التحكم القصيرة.
- **أقل تأخير ممكن** — TCP_NODELAY في كل مكان، تمرير UDP بدون نسخ، قفزة واحدة
  زائدة فقط (اللاعب ← VPS ← أنت).
- **مصادقة تشفيرية** — تحدي/استجابة HMAC-SHA256 على سر مشترك؛ السر لا يمر على
  الشبكة، والريلاي يقبل عميلاً واحداً فقط: أنت.
- **راحة الاستخدام** — اكتشاف تلقائي لسيرفرات الألعاب المحلية، زر مشاركة ينسخ
  العنوان الذي يكتبه اللاعبون، إحصائيات حية لكل نفق، حفظ إعدادات مزدوج مع
  استرجاع تلقائي، وحماية النسخة الواحدة.
- **سيرفر صغير ومحمي** — ملف Go واحد ثابت ~3 ميغا (مكتبة قياسية فقط) يستهلك أقل
  من 10 ميغا رام، يعمل كخدمة systemd محمية.

## هيكل المشروع

| المسار | الوصف |
|---|---|
| `server/` | سيرفر الريلاي بلغة Go (مكتبة قياسية فقط) + اختبارات تكامل |
| `server/deploy/` | سكربت التثبيت `install.sh` + وحدة systemd للـ VPS |
| `client/GameRelay.Core/` | محرك الأنفاق (.NET 9): البروتوكول، إعادة الاتصال، ناقل UDP، مثبّت SSH، كاشف الألعاب |
| `client/GameRelay.App/` | تطبيق سطح المكتب WinUI 3 (Windows) |
| `client/GameRelay.Cli/` | عميل CLI متعدد المنصات (Linux/macOS/Windows) |
| `client/GameRelay.SmokeTest/` | اختبار نهاية-إلى-نهاية (عميل حقيقي ضد سيرفر حقيقي) |
| `scripts/` | سكربتات البناء |
| `docs/` | دليل الإعداد، مواصفة البروتوكول، مرجع منافذ الألعاب |

## البداية السريعة

تحتاج VPS لينكس — **طبقة Oracle Cloud المجانية (Always Free) تكفي ومجانية للأبد**.
الشرح الكامل (إنشاء الجهاز والشبكة والجدار) في **[docs/SETUP.md](docs/SETUP.md)**.

### الأسهل: دع تطبيق Windows يفعلها (v1.3)

1. ابنِ أو حمّل تطبيق Windows وشغّله.
2. اضغط **Set up server** ← اختر **Automatic** ← أدخل عنوان الـ VPS واختر ملف
   مفتاح SSH. التطبيق يثبّت الريلاي، يفتح الجدار الداخلي، ويعبّئ السر تلقائياً.
3. افتح جدار مزوّد السحابة (الجزء الوحيد غير القابل للأتمتة — المعالج يعرض
   القواعد) ثم **Add tunnel** / **Scan for games**. اللاعبون يدخلون
   `عنوان-VPS:المنفذ`.

### يدوياً (أي نظام)

على الـ VPS (أوبنتو، بصلاحيات root):

```bash
scp dist/gamerelay-server-linux-* server/deploy/* user@your-vps:~/gamerelay/
ssh user@your-vps
cd gamerelay && sudo bash install.sh     # سيطبع السر المولّد
```

ثم وجّه عميلاً إليه — تطبيق Windows، أو عميل Linux:

```bash
sudo bash install.sh                     # من حزمة Linux
gamerelay setup                          # العنوان، المنفذ، السر
gamerelay preset "minecraft java"
gamerelay run
```

## ملاحظات أمنية

- الريلاي يقبل عميلاً واحداً موثّقاً فقط (أنت). أي اتصال جديد يستبدل الجلسة القديمة.
- المصادقة تحدي/استجابة HMAC-SHA256، فإعادة إرسال مصافحة ملتقطة لا تنفع (الـ nonce
  يتغير مع كل اتصال)، والسر لا يعبر الشبكة.
- حركة الأنفاق نفسها **غير مشفرة** من طرف الريلاي (الألعاب تشفّر بنفسها). لا تمرر
  لوحات إدارة غير مشفرة.
- معالج SSH يستخدم مفتاحك محلياً للتثبيت؛ ملف المفتاح لا يغادر جهازك أبداً.
- أبقِ `config.json` بصلاحيات `0600` وعامل السر كأنه كلمة مرور.
